# 号码清单补地址层级

## 适用

- 用户给号码、宽带号码或接入号清单，要求回填标准装机地址所属指定层级地址 ID/名称。
- 用户说“几级地址”“地址层级”“装机地址上级地址”“所属地址”，且业务来源仍是号码的标准装机地址。

## 不适用

- 只需要主表已有装机地址字段：先看主表字段。
- 非标准地址层级、用户自定义片区/网格口径、或不是从标准装机地址上卷：先确认来源。

## 主表与补表

| 角色 | 表 | 用途 |
|---|---|---|
| 驱动表 | 用户附件号码清单 | 保留原始序号和号码 |
| 主表 | 069 全业务资料表 | 按号码和账期取 `serv_id/serv_addr_id` |
| 补表 | 079 地址维表 `zone_gz_yz.dwd_yz_addr_final` | 装机地址层级上卷 |

## 通用上卷口径

先把 069 的标准装机地址定位到 079，再按目标 `grade` 上卷。

```text
069.serv_addr_id
  -> 079.id                 -- 标准装机地址，通常 grade=10
  -> parentid / addr_id_*    -- 按目标层级上卷
  -> 079.id (grade=${target_grade})
```

关键写法：

- 起点 JOIN 固定写 `069.serv_addr_id = cast(addr.id as string)`。
- 最终层级 JOIN 必须带 `target.grade = ${target_grade}`，避免取错层级。
- 如果 079 已有目标层级快捷字段（如 `addr_id_7`），可优先用快捷字段二次关联。
- 如果没有快捷字段，按 `parentid` 逐级自关联到目标层级；不要把某个案例里的 9 级、7 级、5 级写死成唯一规则。

## 常见路径示例

### 9 级地址

```text
069.serv_addr_id
  -> 079.id
  -> 079.parentid
  -> 079.id (grade=9)
```

关键写法：

- `069.serv_addr_id = cast(addr10.id as string)`。
- `cast(addr10.parentid as string) = cast(addr9.id as string)`。
- 输出 9 级地址名称时第二次关联必须加 `addr9.grade=9`。

### 7 级地址

```text
069.serv_addr_id
  -> 079.id
  -> 079.addr_id_7
  -> 079.id (grade=7)
```

关键写法：

- `069.serv_addr_id = cast(addr.id as string)`。
- `cast(addr.addr_id_7 as string) = cast(addr7.id as string)`。
- 输出 7 级地址名称时第二次关联必须加 `addr7.grade=7`。

### 5 级地址

```text
serv_addr_id
  -> addr10.id
  -> addr10.addr_id_6
  -> addr6.id
  -> addr6.parentid
  -> addr5.id
```

关键写法：

- 先用主表 `serv_addr_id` 关联 079 的 `id`，取 `addr_id_6`。
- 再用 `addr_id_6` 关联 079 的 `id`，取该 6 级地址行的 `parentid` 作为 5 级地址 ID。
- 最后用 `parentid` 关联 079 的 `id`，取 `addr` 作为 5 级地址名称；建议加 `addr5.grade=5`。

## 硬规则

- 地址 ID 关联统一转字符。
- 禁止默认写 `cast(069.serv_addr_id as decimal(24,0)) = addr.id`，长地址 ID 或异常字符可能导致转换失败或漏数。
- 附件驱动需核对输入行数、069 命中行数、地址命中行数、输出行数。
- 目标层级来自用户需求；用户只说“上级地址”但未说几级时，先追问或输出候选层级，不要猜。

## 自检

- 核对附件输入行数、069 命中行数、079 两次关联命中行数、输出行数。
- 样例检查最终地址名称关联是否带目标 `grade`；涉及多级上卷时检查每一级 `parentid` 或快捷字段是否符合预期。
- 对比 `cast(serv_addr_id as decimal)` 与转字符关联的命中数差异（应为 0 漏数）。
