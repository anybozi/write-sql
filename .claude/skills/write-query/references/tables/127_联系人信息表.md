# 联系人信息表

- **Hive 表名**：`dws_crm_cust.dws_contacts_info`
- **用途**：回填联系人姓名、家庭电话、办公电话、手机和状态时间等联系人资料字段。
- **典型入口**：客户清单已有 `PARTY_ID`，并先通过 126 客户联系人关系表取得 `contact_id`。
- **不要混用**：049 欠费日清单中的联系人字段只适用于欠费推送场景，不是通用客户联系人资料来源。

## 字段说明

| 字段 | 字段类型 | 字段含义 |
|------|----------|----------|
| PARTY_ID |  | 参与方标识 / 客户相关主体标识 |
| contact_id |  | 联系人标识 |
| contact_name |  | 联系人姓名 |
| home_phone |  | 家庭电话 |
| office_phone |  | 办公电话 |
| mobile_phone |  | 手机号码 |
| status_date |  | 状态时间 |
| city_id |  | 地市标识；广州固定 `200` |

## 常用 SQL

### 客户清单回填联系人信息

```sql
select
  a.*,
  c.contact_name,
  c.home_phone,
  c.office_phone,
  c.mobile_phone,
  c.status_date
from seed_customer a
left join (
  select distinct
    cust_id,
    contact_id
  from dws_crm_cust.dws_cust_contact_info_rel
  where city_id = 200
) r
  on a.cust_id = r.cust_id
left join (
  select distinct
    PARTY_ID,
    contact_id,
    contact_name,
    home_phone,
    office_phone,
    mobile_phone,
    status_date
  from dws_crm_cust.dws_contacts_info
  where city_id = 200
) c
  on a.PARTY_ID = c.PARTY_ID
 and r.contact_id = c.contact_id
;
```

## 风险与审计

- 默认 `LEFT JOIN` 保留客户清单，联系人没有则输出空字段。
- 同一客户可能多个联系人；默认输出联系人明细多行。若要求一客户一行，需先确认按 `status_date` 取最新、主联系人、电话聚合或其它去重口径。
- 输出手机号、家庭电话、办公电话属于敏感联系方式，交付前按需求确认是否需要脱敏。
