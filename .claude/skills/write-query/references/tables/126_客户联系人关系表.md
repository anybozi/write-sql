# 客户联系人关系表

- **Hive 表名**：`dws_crm_cust.dws_cust_contact_info_rel`
- **用途**：按客户标识 `cust_id` 找联系人标识 `contact_id`，作为客户清单回填联系人信息的第一跳。
- **典型入口**：用户附件客户清单、客户实体表或其它已圈定客户集合中的 `cust_id`。
- **下游补表**：联系人姓名、电话、状态时间等字段在 127 联系人信息表。

## 字段说明

| 字段 | 字段类型 | 字段含义 |
|------|----------|----------|
| cust_id |  | 客户标识 |
| contact_id |  | 联系人标识 |
| city_id |  | 地市标识；广州固定 `200` |

## 常用 SQL

### 客户清单补联系人标识

```sql
select
  a.*,
  r.contact_id
from seed_customer a
left join (
  select distinct
    cust_id,
    contact_id
  from dws_crm_cust.dws_cust_contact_info_rel
  where city_id = 200
) r
  on a.cust_id = r.cust_id
;
```

## 风险与审计

- 默认 `LEFT JOIN` 保留客户清单，未命中联系人时 `contact_id` 为空。
- 同一 `cust_id` 可能对应多个 `contact_id`，默认输出联系人明细多行；若要一客户一行，需确认取最新联系人、主联系人或聚合电话等口径。
- 补表前后核对客户清单行数、命中客户数，以及按 `cust_id` 是否放大。
