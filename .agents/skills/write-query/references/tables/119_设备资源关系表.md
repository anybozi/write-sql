# 设备资源关系表

- **Hive 表名 / 生产现网名**：`ads_yz_prod_res_inst_rel_final`
- **用途**：按服务标识回填设备资源字段，例如设备名称、设备类型、购买方式、机身号、数量。
- **典型入口**：用户提供号码清单或 `serv_id` 清单，要求补“设备名称 / 设备类型 / 购买方式 / 机身号 / 数量”等字段。

## 字段说明

| 字段 | 字段含义 | 说明 |
|------|----------|------|
| serv_id | 服务标识 | 与 069 或用户附件中的 `serv_id` 关联 |
| mkt_res_name | 设备名称 | 需求口语“设备名称” |
| res_type | 设备类型 | 需求口语“设备类型”；是否需要中文名按需求确认 |
| property_type_name | 购买方式 | 需求口语“购买方式” |
| eqpt_sn | 机身号 | 设备序列 / 机身号 |
| mkt_res_num | 数量 | 设备数量 |

## 常用链路

### 附件号码 / 服务清单补设备资源字段

```sql
-- 附件已有 serv_id 时，可直接关联设备资源表。
select
  seed.serv_id,
  seed.acc_nbr,
  res.mkt_res_name,
  res.res_type,
  res.property_type_name,
  res.eqpt_sn,
  res.mkt_res_num
from tmp_seed seed
left join ads_yz_prod_res_inst_rel_final res
  on seed.serv_id = res.serv_id;
```

附件只有接入号时，先用 069 按账期补 `serv_id`，再关联本表。

## JOIN 与风险

- JOIN 键：`serv_id`。
- 本表可能同一 `serv_id` 多条设备资源记录；默认保留设备明细多行。若用户要求一号一行，需先确认聚合方式。
- 回填附件清单时必须核对：附件行数、补 `serv_id` 命中数、JOIN 设备表后的行数。
- 本表只承载设备资源字段，不用于判断入网订单、套餐在档、299 套餐口径或终端装维成本。
