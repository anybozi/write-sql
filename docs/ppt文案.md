# 项目名称

## AI自然语言取数能力底座

### 面向CDAP业务数据的智能SQL生成能力建设

---

## 第1页｜项目定位

### 打造面向业务取数场景的AI能力底座

本项目聚焦建设一套面向 CDAP 业务数据的自然语言取数能力底座。

通过沉淀业务取数流程、主表路由规则、指标口径、字段映射和 SQL 校验机制，将支撑人员长期依赖经验完成的取数工作，转化为 AI 可理解、可生成、可解释、可校验的标准化能力。

项目目标是先打牢“自然语言转 SQL”的底层能力，为后续业务门户、企业微信机器人、自助取数应用等多类场景提供可复用、可集成的能力支撑。

---

## 第2页｜项目背景

### 数据需求增长快，传统人工取数支撑压力持续增加

随着业务经营数字化程度不断提升，各部门对数据的依赖越来越强。日常经营分析、指标跟踪、营销支撑、专项核查等场景中，取数需求频繁发生。

但在实际支撑过程中，业务需求往往存在表达不标准、口径不清晰、字段不明确等问题。支撑人员需要反复沟通需求、确认口径、选择主表、补齐字段并编写 SQL。

当前取数工作整体依赖个人经验，效率和质量存在不稳定性，也给数据支撑人员带来较大重复性工作压力。

---

## 第3页｜核心痛点

### 当前取数工作存在“三难一重”

**需求收口难**
业务侧通常用自然语言描述需求，表达方式不统一，支撑人员需要反复澄清“查什么、查谁、按什么时间、出什么结果”。

**口径判断难**
同一个业务词可能对应不同口径，例如入网、到达、在网、出账、拆机、续约等，不同口径会影响主表、字段和过滤条件。

**表字段选择难**
CDAP 表多、字段多、历史经验分散，主表选择、字段映射、补表路径很大程度依赖熟手经验。

**重复工作重**
大量常规取数需求具有相似流程，但每次仍需要人工重新理解、编写和检查 SQL，消耗支撑人员大量时间。

---

## 第4页｜项目方案

### 构建“自然语言 → 标准SQL”的智能生成流程

本项目通过“AI技能流程 + 业务知识库 + SQL规则体系”的方式，构建面向 CDAP 取数场景的自然语言转 SQL 能力。

整体流程包括：

1. 自然语言需求拆解
2. 关键条件自动澄清
3. 主表路由判断
4. 指标口径识别
5. 字段映射与补表判断
6. Hive SQL 自动生成
7. SQL 规则校验与风险提示
8. 自检 SQL 输出

通过以上流程，让 AI 不只是“写 SQL”，而是按照支撑人员真实取数逻辑，完成从需求理解到 SQL 生成、从口径说明到结果校验的全过程辅助。

---

## 第5页｜核心能力

### 把支撑人员经验沉淀为可复用的AI能力

**需求收口能力**
将业务自然语言转化为 SQL 所需的关键要素，包括查询对象、时间口径、统计方式、输出字段和限制条件。

**主表路由能力**
根据业务事实判断应从哪张 CDAP 主表出发，减少因字段名相似导致的误选表问题。

**口径识别能力**
沉淀常见指标和业务动作口径，确保 SQL 过滤条件与实际业务定义一致。

**字段补表能力**
当主表字段不足时，按照规则判断是否需要补维表、补事实表，并识别 JOIN 风险。

**SQL生成能力**
根据已确认的主表、字段、口径和过滤条件，自动生成 Hive SQL。

**结果校验能力**
在 SQL 生成后同步输出字段来源、口径说明、风险提示和自检 SQL，提升结果可解释性和可校验性。

## 第6页｜本期建设重点

### 聚焦自然语言取数能力底座，打牢“生成准确”的基础能力

本期项目重点围绕 CDAP 业务数据取数场景，建设一套可复用的自然语言转 SQL 能力底座。

通过沉淀支撑人员在日常取数中的经验规则，将自然语言需求拆解、主表路由、指标口径识别、字段映射、补表判断、SQL 生成和结果校验等关键环节标准化、流程化、知识化。

本期重点建设内容包括：

* 自然语言需求拆解流程
* CDAP 主表路由规则
* 指标口径识别规则
* 字段映射与补表判断规则
* Hive SQL 自动生成流程
* SQL 规则校验与自检机制
* 表路由、字段补表、业务口径等知识库沉淀

本期目标是先把自然语言转 SQL 的准确性、可解释性和可校验性打牢，为后续业务侧自然语言取数能力扩展提供基础支撑。

---

## 第7页｜应用路径

### 能力底座先行，支撑后续场景扩展

项目应用路径可分为两个层次推进。

**第一层：内部支撑场景验证**

先面向内部数据支撑人员使用，辅助完成常规取数需求的 SQL 初稿生成、口径说明和结果检查。

支撑人员从“手工编写 SQL”逐步转变为“审核 AI 生成结果”，在提升需求响应效率的同时，持续完善表路由、指标口径、字段补表和校验规则，推动经验沉淀和能力标准化。

**第二层：能力开放与场景复用**

当能力底座逐步稳定后，可作为统一的自然语言转 SQL 能力组件，支撑后续多类业务取数入口复用。

未来可支撑的方向包括：

* 业务门户自然语言取数
* 企业微信机器人问数
* 数据服务平台智能查询
* 常用指标智能问答
* 业务数据需求模板化生成

通过“能力底座先行、应用场景复用”的方式，逐步支撑业务侧从“提需求等取数”向“自然语言辅助取数”转变。

---

## 第8页｜预期价值

### 从“人工经验取数”走向“AI辅助支撑”

本项目落地后，预计可带来以下价值：

**对支撑人员**
减少重复性 SQL 编写工作，提升常规取数需求响应效率，推动支撑人员从“手工写 SQL”向“审核 AI 结果、治理数据口径”转变。

**对业务部门**
为后续自然语言辅助取数提供基础能力，降低业务人员使用数据的门槛。

**对数据治理**
沉淀统一的表路由、指标口径、字段补表和校验规则，减少经验分散和口径不一致问题。

**对组织能力**
推动数据支撑模式从“个人经验驱动”向“知识库 + AI 协同驱动”转变。

**对后续应用**
为门户、机器人、自助取数平台等 AI 数据应用提供可复用、可集成的 SQL 生成能力底座。

预期成效目标：

* 常规 SQL 初稿生成效率提升 50%以上
* 需求澄清和口径确认时间减少 30%以上
* 新人上手周期明显缩短
* SQL 口径偏差、字段误用、漏加过滤等问题逐步减少

总结一句话：

**本项目不是单点取数工具，而是打造支撑未来业务自然语言取数的 AI 能力底座。**
